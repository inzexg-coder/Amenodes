# `LanguageManager`

## ОПИСАНИЕ

`LanguageManager` – центральный класс системы интернационализации (i18n) приложения Amenodes. Реализует управление переводами, переключение языков, подписку на изменения и глубокое слияние (deep merge) локализационных словарей.
Класс обеспечивает единую точку доступа ко всем переводным строкам приложения, включая как базовые UI-компоненты, так и специфичные для узлов переводы, загружаемые динамически.

**Архитектурные особенности:**

- Класс реализован как синглтон – экспортируется единственный экземпляр `i18n`.
- Поддерживает подписку на изменения языка.
- Обеспечивает ленивую загрузку и слияние переводов узлов через механизм `setNodeTranslations`.
- Использует `localStorage` для сохранения выбранного языка между сессиями.
- Автоматически определяет язык браузера при первом запуске.

## ЗАВИСИМОСТИ

```javascript
import { en } from './locales/en.js';
import { ru } from './locales/ru.js';
```

| Импорт | Назначение |
|--------|------------|
| `en`, `ru` | Базовые словари языков (UI, общие строки, сообщения об ошибках). |

# КОНСТАНТЫ

## BASE_LOCALES

```javascript
const BASE_LOCALES = {
  ru: { name: 'Русский', nativeName: 'Русский', translations: ru },
  en: { name: 'English', nativeName: 'English', translations: en }
};
```

Объект, определяющий поддерживаемые языки и их базовые переводы. Структура записи:

| Поле | Тип | Описание |
|------|-----|----------|
| `name` | `string` | Название языка на английском (для отображения в меню). |
| `nativeName` | `string` | Название языка на родном языке. |
| `translations` | `object` | Словарь переводов (импортированный из соответствующего файла локали). |

# КЛАСС LANGUAGEMANAGER

## Конструктор

```javascript
constructor()
```

Создаёт новый экземпляр менеджера локализации. Инициализирует:

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.currentLanguage` | `string` | результат `getSavedLanguage()` | Код текущего активного языка (например, `'en'`). |
| `this.listeners` | `Array<Function>` | `[]` | Массив callback-функций, вызываемых при смене языка. |
| `this.nodeTranslations` | `object` | `{ en: {}, ru: {} }` | Хранилище переводов, загруженных из файлов узлов. |
| `this.translations` | `object` | `{}` | Результирующий объединённый словарь (базовый + переводы узлов). |

После инициализации вызывает `this.updateTranslations()` для построения итогового словаря.

## Публичные свойства

| Свойство | Тип | Доступ | Описание |
|----------|-----|--------|----------|
| `currentLanguage` | `string` | чтение (через геттер) | Код текущего языка. |
| `translations` | `object` | чтение | Объединённый словарь всех переводов. |
| `listeners` | `Array<Function>` | чтение | Массив подписчиков на изменения языка. |

# Методы

## setNodeTranslations(translations)

```javascript
setNodeTranslations(translations: object): void
```

Устанавливает переводы, специфичные для узлов (загружаемые из файлов `src/nodes/locales/*/`). 
Выполняет глубокое слияние с существующими переводами узлов, затем перестраивает итоговый словарь и уведомляет всех подписчиков об изменении.

**Параметры:**

- `translations` – объект со структурой:
  ```javascript
  {
    en: { ... },   // переводы на английский для узлов
    ru: { ... }    // переводы на русский для узлов
  }
  ```

- Обновляет `this.nodeTranslations` (глубокое слияние).
- Вызывает `this.updateTranslations()`.
- Вызывает `this.notifyListeners()`.

## updateTranslations()

```javascript
updateTranslations(): void
```

Перестраивает итоговый словарь `this.translations`, выполняя глубокое слияние базовых переводов текущего языка с переводами узлов для этого же языка.

**Алгоритм:**

1. Получает базовые переводы: `BASE_LOCALES[this.currentLanguage]?.translations`.
2. Получает переводы узлов: `this.nodeTranslations[this.currentLanguage]`.
3. Выполняет глубокое слияние через `this.mergeDeep(base, nodes)`.
4. Сохраняет результат в `this.translations`.

**Вызывается автоматически:**
- При инициализации экземпляра.
- При смене языка (`setLanguage`).
- При установке переводов узлов (`setNodeTranslations`).

## mergeDeep(target, source)

```javascript
mergeDeep(target: object, source: object): object
```

Выполняет рекурсивное глубокое слияние двух объектов. Массивы не сливаются – полностью заменяются (в соответствии с ожидаемым поведением для словарей переводов). Функция создаёт новый объект, не мутируя исходные.

**Параметры:**

- `target` – базовый объект (будет скопирован, но не мутирован).
- `source` – объект-источник, значения которого имеют приоритет.

**Алгоритм:**

1. Создаёт поверхностную копию `target` в переменную `result`.
2. Для каждого ключа в `source`:
   - Если значение в `source` является объектом (и не массивом), а также существует в `result`, рекурсивно сливает.
   - Иначе (простое значение или массив) заменяет значение в `result` на значение из `source`.
3. Возвращает `result`.

**Возвращает:** новый объект, являющийся результатом глубокого слияния.

**Пример:**

```javascript
const base = { a: 1, b: { x: 10, y: 20 } };
const source = { b: { y: 30, z: 40 }, c: 3 };
const result = i18n.mergeDeep(base, source);
// result = { a: 1, b: { x: 10, y: 30, z: 40 }, c: 3 }
```

## getSavedLanguage()

```javascript
getSavedLanguage(): string
```

Определяет язык для использования при загрузке приложения. Приоритет:

1. Язык, сохранённый в `localStorage` по ключу `'amenodes_language'` (если он поддерживается в `BASE_LOCALES`).
2. Язык браузера (`navigator.language.split('-')[0]`), если он поддерживается.
3. Английский (`'en'`) как резервный вариант.

**Возвращает:** строку с кодом языка.

## setLanguage(lang)

```javascript
setLanguage(lang: string): boolean
```

Устанавливает новый активный язык. Сохраняет выбор в `localStorage`, перестраивает словарь переводов, уведомляет всех подписчиков и обновляет глобальную переменную `window._currentLanguage`.

**Параметры:**

- `lang` – код языка.

**Возвращает:** 
- `true` – если язык успешно установлен (язык поддерживается).
- `false` – если передан неподдерживаемый код языка.

**Побочные эффекты:**

- Обновляет `this.currentLanguage`.
- Устанавливает `window._currentLanguage = lang`.
- Сохраняет язык в `localStorage.setItem('amenodes_language', lang)`.
- Вызывает `this.updateTranslations()`.
- Вызывает `this.notifyListeners()`.

**Пример:**

```javascript
const success = i18n.setLanguage('ru');
if (success) {
  console.log('Язык переключён на русский');
}
```

## getCurrentLanguage()

```javascript
getCurrentLanguage(): string
```

Возвращает код текущего активного языка.

**Возвращает:** `string` – код языка.

## getAvailableLanguages()

```javascript
getAvailableLanguages(): Array<object>
```

Возвращает массив объектов с информацией о всех поддерживаемых языках.

**Возвращает:** массив объектов со структурой:

```javascript
[
  {
    code: 'en',
    name: 'English',
    nativeName: 'English'
  },
  {
    code: 'ru',
    name: 'Russian',
    nativeName: 'Русский'
  }
]
```

## t(key, params)

```javascript
t(key: string, params: object = {}): string
```

Возвращает локализованную строку для указанного ключа. Поддерживает подстановку параметров в формате `{paramName}`.

**Параметры:**

- `key` – ключ перевода в формате `section.subsection.key` (например, `'common.ok'`, `'toolbar.undo'`).
- `params` – объект с параметрами для подстановки. Ключи в фигурных скобках в строке заменяются на соответствующие значения.

**Алгоритм:**

1. Разбивает `key` по символу `.` на массив ключей.
2. Последовательно обходит объект `this.translations`, извлекая значение по каждому сегменту пути.
3. Если значение не найдено (тип не `string`), возвращает исходный `key`.
4. Если найдено строковое значение, заменяет все вхождения `{param}` на значения из `params`.
5. Если параметр не передан, оставляет `{param}` как есть.

**Возвращает:** локализованную строку или исходный `key` при отсутствии перевода.

**Пример:**

```javascript
i18n.t('common.ok');                    // 'OK' (для английского) или 'ОК' (для русского)
i18n.t('errors.cannotConnect');         // 'Cannot connect' / 'Невозможно соединить'
i18n.t('modal.enterValue', { count: 5 }); // 'Enter value' (параметры не используются в этом ключе)
```

## subscribe(listener)

```javascript
subscribe(listener: Function): Function
```

Добавляет callback-функцию в список подписчиков. Подписчики вызываются при каждом изменении языка (и только при изменении языка, не при каждом обновлении переводов узлов, хотя `setNodeTranslations` также вызывает `notifyListeners`).

**Параметры:**

- `listener` – функция, принимающая два аргумента:
  - `lang` (`string`) – новый код языка.
  - `translations` (`object`) – новый объединённый словарь переводов.

**Возвращает:** функцию отписки (unsubscribe). При её вызове подписчик удаляется из массива `this.listeners`.

**Пример:**

```javascript
const unsubscribe = i18n.subscribe((lang, translations) => {
  console.log(`Язык изменён на ${lang}`);
  updateAllUIStrings();
});

// Позже, когда компонент уничтожается:
unsubscribe();
```

## notifyListeners()

```javascript
notifyListeners(): void
```

Вызывает все зарегистрированные callback-функции, передавая им текущий язык и итоговый словарь переводов. Вызывается автоматически при:

- Смене языка через `setLanguage`.
- Установке переводов узлов через `setNodeTranslations` (вызывает `updateTranslations`, затем `notifyListeners`).

**Защита:** передаёт подписчикам прямые ссылки на `this.currentLanguage` и `this.translations`.

# СТРУКТУРА СЛОВАРЯ ПЕРЕВОДОВ

Базовые переводы (`en.js`, `ru.js`) имеют следующую структуру верхнего уровня:

```javascript
{
  common: { ... },        // Общие строки (OK, Cancel, Save и т.д.)
  toolbar: { ... },       // Кнопки панели инструментов
  nodeMenu: { ... },      // Меню добавления узлов
  contextMenu: { ... },   // Контекстное меню узлов
  optimizations: { ... }, // Оптимизации и бенчмаркинг
  modal: { ... },         // Диалоговые окна
  errors: { ... },        // Сообщения об ошибках
  status: { ... },        // Статусные сообщения
  operations: { ... },    // Уведомления об операциях
  editor: { ... },        // Редактор (кнопки, заголовки)
  templates: { ... },     // Страница библиотеки шаблонов
  profile: { ... },       // Страница профиля пользователя
  moderate: { ... }       // Панель модерации
}
```
Переводы узлов (`src/nodes/locales/*/`) добавляют и/или переопределяют секции:

```javascript
{
  nodes: { ... },              // Названия типов узлов
  nodeDescriptions: { ... },   // Описания узлов
  calcTypes: { ... },          // Типы калькуляторов
  output: { ... },             // Секции выходного узла
  group: { ... },              // Секции группового узла
  map: { ... },                // Секции узла отображения
  confidence: { ... },         // Секции доверительных интервалов
  dataTypes: { ... }           // Названия типов данных
}
```

# ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Получение перевода в компоненте

```javascript
import { i18n, t } from '../i18n/LanguageManager.js';

// Использование функции t (краткая форма)
const okButtonText = t('common.ok');

// Использование экземпляра напрямую
const cancelText = i18n.t('common.cancel');
```

### Реактивное обновление UI при смене языка

```javascript
class MyComponent {
  constructor() {
    this.unsubscribe = i18n.subscribe(() => this.render());
    this.render();
  }
  
  render() {
    this.button.textContent = t('common.save');
    this.label.textContent = t('nodes.output');
  }
  
  destroy() {
    if (this.unsubscribe) this.unsubscribe();
  }
}
```

### Добавление поддержки нового языка

Для добавления нового языка необходимо:

1. Создать файл перевода в `src/i18n/locales/`, например `de.js`.
2. Добавить запись в `BASE_LOCALES`:
   ```javascript
   const BASE_LOCALES = {
     // ... существующие языки
     de: { name: 'German', nativeName: 'Deutsch', translations: de }
   };
   ```
3. Создать директорию `src/nodes/locales/de/` и добавить переводы для узлов.

# ЗАМЕЧАНИЯ

- **Глобальная переменная:** Экземпляр `i18n` также доступен как `window._currentLanguage` (только код языка, не сам экземпляр). Это используется для совместимости с устаревшим кодом.
- **Ленивая загрузка:** Переводы узлов не загружаются автоматически при инициализации – они должны быть явно переданы через `setNodeTranslations` после загрузки манифеста узлов. За это отвечает модуль `src/nodes/registry.js`.
- **Производительность:** `mergeDeep` вызывается при каждом обновлении переводов. При большом количестве узлов (сотни) слияние объектов может создавать заметную нагрузку при инициализации, но не в рантайме.
- **Типизация:** В системе типов (`DataType.js`) для получения локализованных имён используется `graph.getTypeDisplayName(typeKey)`, который, в свою очередь, вызывает `t('dataTypes.${typeKey}')`.
- **Подписка и утечки памяти:** Каждый компонент, создающий DOM-элементы с динамическими переводами (например, `Node.createDOM`), обязан сохранять функцию отписки и вызывать её при уничтожении элемента. В противном случае подписчики будут накапливаться, вызывая утечки памяти.
