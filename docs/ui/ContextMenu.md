# `ContextMenu`

## ОПИСАНИЕ

`ContextMenu` – класс, реализующий контекстное меню для узлов графа. Меню вызывается при клике на манипулятор (handle) узла.
Класс поддерживает динамическое обновление переводов: при смене языка интерфейса меню автоматически перестраивается с корректными локализованными текстами.

**Ключевые особенности:**
- Создание новых узлов с автоматическим соединением к исходному узлу
- Поддержка категорий с вложенными подменю
- Отметка узлов как «важных» (визуальное выделение)
- **Валидация типов перед созданием узла** – узел создаётся и проверяется на совместимость типов ДО добавления в граф
- При ошибке совместимости типов узел не добавляется в граф (предотвращает появление «битых» узлов)

## ЗАВИСИМОСТИ

```javascript
import { NodeFactory } from '../nodes/NodeFactory.js';
import { modal } from './CustomModal.js';
import { i18n, t } from '../i18n/LanguageManager.js';
```

| Импорт | Назначение |
|--------|------------|
| `NodeFactory` | Фабрика для создания экземпляров узлов по их типу (асинхронная, с поддержкой `onCreate`). |
| `modal` | Объект для отображения модальных диалогов (сообщения об ошибках совместимости типов). |
| `i18n`, `t` | Система интернационализации для локализации текстов меню и сообщений об ошибках. |

# КЛАСС CONTEXTMENU

## Конструктор

```javascript
constructor(graph: Graph, renderer: DomRenderer, history: History, viewport: Viewport)
```

Создаёт экземпляр контекстного меню, привязанный к конкретным компонентам приложения.

**Параметры:**

| Имя | Тип | Описание |
|-----|-----|----------|
| `graph` | `Graph` | Экземпляр графа, над которым выполняются операции. |
| `renderer` | `DomRenderer` | Экземпляр рендерера, используемый для обновления отображения после изменений. |
| `history` | `History` | Экземпляр системы истории (undo/redo), используемый для сохранения состояний. |
| `viewport` | `Viewport` | Экземпляр области просмотра, используемый для вычисления позиций новых узлов. |

**Инициализируемые свойства:**

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.graph` | `Graph` | параметр | Ссылка на граф. |
| `this.renderer` | `DomRenderer` | параметр | Ссылка на рендерер. |
| `this.history` | `History` | параметр | Ссылка на систему истории. |
| `this.viewport` | `Viewport` | параметр | Ссылка на область просмотра. |
| `this.currentMenu` | `HTMLElement \| null` | `null` | Текущее отображаемое DOM-элемент меню. |
| `this.currentSourceId` | `number \| null` | `null` | ID узла, для которого открыто меню. |
| `this.currentBaseX` | `number \| null` | `null` | Базовая X-координата для позиционирования новых узлов. |
| `this.currentBaseY` | `number \| null` | `null` | Базовая Y-координата для позиционирования новых узлов. |

- Подписывается на изменения языка через `i18n.subscribe()`. При смене языка текущее открытое меню автоматически перестраивается с новыми переводами.

# Публичные методы

## show(x, y, sourceId)

```javascript
show(x: number, y: number, sourceId: number): void
```

Отображает контекстное меню в указанных экранных координатах для заданного узла-источника.

**Параметры:**

| Имя | Тип | Описание |
|-----|-----|----------|
| `x` | `number` | Горизонтальная координата (в пикселях, относительно окна браузера) для левого верхнего угла меню. |
| `y` | `number` | Вертикальная координата (в пикселях, относительно окна браузера) для левого верхнего угла меню. |
| `sourceId` | `number` | Идентификатор узла, для которого вызывается меню (все операции создания узлов будут создавать связь от этого узла). |

**Структура меню:**
1. Пункт "Создать вывод и соединить" (OutputNode)
2. Разделитель
3. Категории узлов (с подменю)
4. Обычные узлы
5. Разделитель
6. Пункты "Выделить ВАЖНЫЙ узел" / "Снять выделение ВАЖНОГО"

**Пример:**

```javascript
const menu = new ContextMenu(graph, renderer, history, viewport);
menu.show(200, 300, 5); // открыть меню для узла с ID=5 в позиции (200, 300)
```

## getNodeScreenPosition(sourceNode)

```javascript
getNodeScreenPosition(sourceNode: Node): { x: number, y: number }
```

Вычисляет экранные координаты для позиционирования новых узлов относительно существующего узла.

**Параметры:**

| Имя | Тип | Описание |
|-----|-----|----------|
| `sourceNode` | `Node` | Узел, относительно которого вычисляется позиция. |

**Алгоритм:**
- Если узел существует и доступен `viewport`, вычисляет позицию с учётом смещения и масштаба:
  ```
  screenX = node.x + offset.x / zoom + 140
  screenY = node.y + offset.y / zoom + 40
  ```
- Если узел не существует или `viewport` недоступен, возвращает координаты центра области просмотра.

**Возвращает:** объект с полями `x` и `y` (экранные координаты в пикселях).

## createAndConnectWithCheck(nodeType, x, y, sourceId, extraOptions)

```javascript
async createAndConnectWithCheck(nodeType: string, x: number, y: number, sourceId: number, extraOptions: object = {}): Promise<void>
```

**Основной метод создания узла с проверкой совместимости типов.** 
Узел создаётся временно (без добавления в граф), проверяется возможность соединения, и только при успешной проверке добавляется в граф.

**Параметры:**

| Имя | Тип | Описание |
|-----|-----|----------|
| `nodeType` | `string` | Строковый идентификатор типа узла (например, `'number'`, `'output'`). |
| `x` | `number` | Горизонтальная координата для нового узла (в мировых координатах). |
| `y` | `number` | Вертикальная координата для нового узла (в мировых координатах). |
| `sourceId` | `number` | Идентификатор узла-источника, от которого будет создана связь. |
| `extraOptions` | `object` | Дополнительные параметры, передаваемые в конструктор узла. |

**Алгоритм:**

1. Получает узел-источник по `sourceId`.
2. **Временно создаёт узел** через `NodeFactory.createNode()` с временным ID (`-999`), **НЕ добавляя его в граф**.
3. Если узел не создался (пользователь отменил создание) – завершает работу.
4. Проверяет возможность соединения через `graph.canConnect(sourceId, tempNode.id)`.
5. **Если соединение невозможно:**
   - Получает локализованные имена типов через `graph.getTypeDisplayName()`
   - Показывает модальное окно с сообщением об ошибке: `"Cannot connect: Array → Number"`
   - Узел **отбрасывается** (не добавляется в граф)
6. **Если соединение возможно:**
   - Добавляет узел в граф через `graph.addNode(tempNode)`
   - Создаёт связь через `graph.addEdge(sourceId, tempNode.id, 'main')`
   - Вызывает `finishNodeCreation()` для пересчёта и сохранения

**Важно:** Этот метод предотвращает появление в графе «битых» узлов – узлов, созданных при попытке несовместимого соединения.

## finishNodeCreation()

```javascript
finishNodeCreation(): void
```

Завершает процесс создания узла, выполняя необходимые обновления.

**Действия:**
- Вызывает `graph.reevaluateAll()` – пересчёт значений всех узлов.
- Вызывает `graph.updateAllOutputs()` – обновление отображаемых данных.
- Вызывает `renderer.render()` – перерисовка интерфейса.
- Вызывает `history.save()` – сохранение состояния для undo/redo.
- Обновляет счётчики узлов и связей в статус-баре.

## toggleImportant(node, important)

```javascript
toggleImportant(node: Node, important: boolean): void
```

Устанавливает или снимает флаг важности узла.

**Параметры:**

| Имя | Тип | Описание |
|-----|-----|----------|
| `node` | `Node` | Целевой узел. |
| `important` | `boolean` | `true` для отметки как важного, `false` для снятия отметки. |

**Действия:**
- Изменяет свойство `node.important`.
- Вызывает `renderer.updateNodeClass(node)` для обновления CSS-класса.
- Вызывает `renderer.render()` для перерисовки.
- Вызывает `history.save()` для сохранения состояния.

## close()

```javascript
close(): void
```

Закрывает текущее открытое контекстное меню.

**Действия:**
- Удаляет DOM-элемент меню из документа.
- Устанавливает `this.currentMenu = null` и `this.currentSourceId = null`.

# Приватные методы

## addMenuItem(menu, text, onClick)

Добавляет стандартный пункт меню.

**Параметры:**

- `text` – текст пункта меню.
- `onClick` – функция, вызываемая при выборе пункта (после чего меню автоматически закрывается).

Создаёт элемент меню с вложенным подменю.

**Параметры:**
- `title` – текст заголовка подменю.
- `items` – массив элементов подменю.
- `onSelect` – колбэк, вызываемый при выборе пункта подменю.

# ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Базовое использование

```javascript
import { ContextMenu } from './ui/ContextMenu.js';

const contextMenu = new ContextMenu(graph, renderer, history, viewport);
contextMenu.show(500, 400, 3);
```

### Интеграция с рендерером

```javascript
// В DomRenderer.js
showMenu(x, y, sourceId) {
  if (!this.contextMenu) {
    this.contextMenu = new ContextMenu(this.graph, this, this.history, this.viewport);
  }
  this.contextMenu.show(x, y, sourceId);
}
```

### Закрытие меню извне

```javascript
window.addEventListener('keydown', (e) => {
  if (e.key === 'Escape') {
    contextMenu.close();
  }
});
```

## ЗАМЕЧАНИЯ

- **Валидация типов:** Ключевое улучшение – узел создаётся и проверяется ДО добавления в граф. При несовместимости типов узел не остаётся в графе.
- **Локализация:** Меню автоматически перестраивается при смене языка. Сохраняется `currentSourceId`, позиция меню восстанавливается по сохранённым координатам.
- **Асинхронность:** Все операции создания узлов асинхронны (`async/await`) для поддержки узлов с `onCreate` (например, `ConstantNode`).
- **Сообщения об ошибках:** При несовместимости типов показывается модальное окно с локализованным сообщением, например: `"Cannot connect: Array → Number"`.
- **Позиционирование:** Меню позиционируется абсолютно относительно окна браузера (использует `position: fixed` в CSS).
- **Закрытие:** После выбора любого пункта меню (включая подменю) меню закрывается автоматически.
- **Навигация:** Класс не управляет фокусом клавиатуры – навигация по меню возможна только мышью.
