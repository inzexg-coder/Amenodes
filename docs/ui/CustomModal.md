# `CustomModal`

## ОПИСАНИЕ

`CustomModal` – система модальных диалоговых окон для Amenodes, предоставляющая замену стандартным функциям `alert`, `confirm` и `prompt` с кастомизированным визуальным оформлением. 
Компонент реализует асинхронный интерфейс на основе `Promise`, что позволяет корректно интегрироваться с асинхронным кодом приложения.

Класс отвечает за:

- создание и стилизацию модальных окон;
- отображение уведомлений (`alert`);
- отображение диалогов подтверждения (`confirm`);
- отображение диалогов с текстовым вводом (`prompt`);
- автоматическое внедрение CSS-стилей в документ;
- управление анимацией появления/исчезновения;
- блокировку взаимодействия с основным интерфейсом при открытом модальном окне.

**Важно:** `CustomModal` переопределяет глобальные функции `window.alert`, `window.confirm` и `window.prompt` для единообразного использования во всём приложении.

## ЗАВИСИМОСТИ

```javascript
import('../i18n/LanguageManager.js').then(({ t }) => { ... })
```

| Импорт | Назначение |
|--------|------------|
| `i18n`, `t` | Система интернационализации для локализации текста кнопок и заголовков. Импортируются динамически для предотвращения циклических зависимостей. |

**Примечание:** зависимости импортируются динамически внутри методов, а не в заголовке файла, так как `CustomModal` используется модулями, которые сами зависят от `i18n`.

# КЛАСС CUSTOMMODAL

## Конструктор

```javascript
constructor()
```

Создаёт экземпляр модального диалога. Инициализирует:

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.modal` | `HTMLElement` | `null` | Ссылка на текущий DOM-элемент модального окна (или `null`, если окно закрыто). |

Вызывает `this.createStyles()` для внедрения CSS-стилей в документ (если они ещё не были добавлены).

## createStyles()

```javascript
createStyles(): void
```

Создаёт и внедряет в `<head>` документа CSS-стили для модальных окон. Стили включают:

| Селектор / Класс | Назначение |
|------------------|------------|
| `.custom-modal-overlay` | Затемняющий фон с размытием (backdrop-filter). |
| `.custom-modal` | Основной контейнер окна с градиентным фоном и анимацией. |
| `.custom-modal-header` | Верхняя панель с заголовком. |
| `.custom-modal-body` | Область содержимого. |
| `.custom-modal-input` | Стилизованное поле ввода. |
| `.custom-modal-buttons` | Контейнер для кнопок. |
| `.custom-modal-btn-*` | Стили для кнопок (primary, secondary, danger). |

**Поведение:** если в документе уже существует элемент с идентификатором `custom-modal-styles`, повторное внедрение не происходит.

**Примечание:** стили внедряются однократно при создании первого экземпляра `CustomModal` и не удаляются в течение жизни приложения.

## alert(message, title)

```javascript
async alert(message: string, title?: string): Promise<void>
```

Отображает модальное окно с сообщением и одной кнопкой "OK". Является асинхронной заменой стандартного `window.alert`.

**Параметры:**

- `message` – текст сообщения, отображаемый в теле окна.
- `title` – заголовок окна (опционально). Если не указан, используется локализованная строка `modal.notification`.

**Возвращает:** `Promise`, который разрешается после закрытия окна пользователем.

**Алгоритм:**

1. Динамически импортирует модуль `i18n/LanguageManager.js`.
2. Получает заголовок через `t('modal.notification')`, если `title` не передан.
3. Вызывает `this.showModal` с конфигурацией для `alert`.
4. Возвращает `Promise`, разрешающийся при вызове колбэка кнопки.

**Пример:**

```javascript
import { modal } from './ui/CustomModal.js';

await modal.alert('Операция выполнена успешно');
await modal.alert('Неизвестная ошибка', 'Ошибка');
```

## confirm(message, title)

```javascript
async confirm(message: string, title?: string): Promise<boolean>
```

Отображает модальное окно с сообщением и двумя кнопками: "OK" (danger) и "Cancel" (secondary). Является асинхронной заменой стандартного `window.confirm`.

**Параметры:**

- `message` – текст сообщения.
- `title` – заголовок окна (опционально). Если не указан, используется локализованная строка `modal.confirm`.

**Возвращает:** `Promise<boolean>` – `true`, если пользователь нажал "OK", иначе `false`.

**Алгоритм:**

1. Динамически импортирует модуль `i18n`.
2. Получает заголовок через `t('modal.confirm')`, если `title` не передан.
3. Вызывает `this.showModal` с конфигурацией, содержащей две кнопки.
4. Возвращает `Promise`, разрешающийся с `true` при нажатии "OK" и `false` при нажатии "Cancel".

**Пример:**

```javascript
const shouldDelete = await modal.confirm('Вы действительно хотите удалить этот элемент?');
if (shouldDelete) {
  // выполнить удаление
}
```

## prompt(message, defaultValue, title)

```javascript
async prompt(message: string, defaultValue?: string, title?: string): Promise<string | null>
```

Отображает модальное окно с сообщением, полем ввода текста и кнопками "OK" / "Cancel". Является асинхронной заменой стандартного `window.prompt`.

**Параметры:**

- `message` – текст сообщения, отображаемый над полем ввода.
- `defaultValue` – значение по умолчанию для поля ввода (опционально, по умолчанию `''`).
- `title` – заголовок окна (опционально). Если не указан, используется локализованная строка `modal.enterValue`.

**Возвращает:** `Promise<string | null>` – введённое пользователем значение или `null`, если пользователь нажал "Cancel" или закрыл окно.

**Особенности:**

- Поле ввода автоматически получает фокус при открытии окна (с задержкой 50 мс для корректного рендеринга).
- Текст в поле ввода автоматически выделяется (`input.select()`).
- Нажатие клавиши `Enter` в поле ввода эквивалентно нажатию "OK".

**Пример:**

```javascript
const value = await modal.prompt('Введите значение:', '0');
if (value !== null) {
  const num = parseFloat(value);
  console.log(`Введено число: ${num}`);
}
```

## showModal(options)

```javascript
showModal(options: ModalOptions): void
```

Базовый метод отображения модального окна. Создаёт DOM-структуру, добавляет обработчики событий и отображает окно. Предыдущее открытое окно (если есть) закрывается перед созданием нового.

**Параметры:** объект со следующими полями:

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `title` | `string` | Да | Заголовок окна. |
| `body` | `string` | Нет* | Текстовое содержимое тела окна. |
| `bodyElement` | `HTMLElement` | Нет* | DOM-элемент для вставки в тело окна. |
| `buttons` | `Array<ModalButton>` | Да | Массив кнопок. |
| `onShow` | `Function` | Нет | Колбэк, вызываемый после отображения окна. |

\* Должен быть указан либо `body`, либо `bodyElement`.

**Структура ModalButton:**

| Поле | Тип | Описание |
|------|-----|----------|
| `text` | `string` | Текст кнопки. |
| `type` | `'primary' \| 'secondary' \| 'danger'` | Стиль кнопки. |
| `callback` | `Function` | Функция, вызываемая при нажатии. |

- Удаляется предыдущее модальное окно (вызов `this.close()`).
- Создаётся overlay с затемнением фона.
- Создаётся модальное окно с заголовком, телом и кнопками.
- Добавляется обработчик закрытия при клике на overlay.
- Если передан `onShow`, вызывается с задержкой 10 мс.
- Ссылка на созданный overlay сохраняется в `this.modal`.

**Пример вызова:**

```javascript
this.showModal({
  title: 'Подтверждение',
  body: 'Вы уверены?',
  buttons: [
    { text: 'Нет', type: 'secondary', callback: () => { this.close(); resolve(false); } },
    { text: 'Да', type: 'danger', callback: () => { this.close(); resolve(true); } }
  ]
});
```

## close()

```javascript
close(): void
```

Закрывает текущее открытое модальное окно (если оно существует). Удаляет DOM-элемент overlay из документа и обнуляет ссылку `this.modal`.

Не вызывает колбэки кнопок – эти колбэки уже должны быть вызваны перед закрытием.

# ГЛОБАЛЬНЫЙ ЭКЗЕМПЛЯР

```javascript
export const modal = new CustomModal();
```

Приложение Amenodes экспортирует единственный экземпляр класса `CustomModal`. Все компоненты должны использовать именно этот экземпляр, а не создавать собственный.

## ИНТЕГРАЦИЯ С ГЛОБАЛЬНЫМИ ФУНКЦИЯМИ

В `main.js` производится переопределение стандартных функций:

```javascript
window.alert = (msg) => { modal.alert(msg); };
window.confirm = (msg) => modal.confirm(msg);
window.prompt = (msg, def) => modal.prompt(msg, def);
```

Это гарантирует, что любой код (включая сторонние библиотеки) будет использовать кастомные модальные окна вместо нативных.

# ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Простое уведомление

```javascript
import { modal } from '../ui/CustomModal.js';

await modal.alert('Данные успешно сохранены');
```

### Диалог подтверждения перед удалением

```javascript
const confirmed = await modal.confirm('Удалить все узлы?');
if (confirmed) {
  graph.nodes.forEach(node => graph.removeNode(node.id));
}
```

### Запрос значения с предзаполнением

```javascript
const name = await modal.prompt('Введите имя узла:', 'Новый узел');
if (name && name.trim()) {
  node.title = name.trim();
}
```

### Кастомное содержимое

```javascript
const container = document.createElement('div');
container.innerHTML = `
  <label>Выберите опцию:</label>
  <select id="optionSelect">
    <option value="1">Опция 1</option>
    <option value="2">Опция 2</option>
  </select>
`;

modal.showModal({
  title: 'Настройки',
  bodyElement: container,
  buttons: [
    { text: 'Отмена', type: 'secondary', callback: () => modal.close() },
    { 
      text: 'Применить', 
      type: 'primary', 
      callback: () => {
        const selected = document.getElementById('optionSelect').value;
        console.log('Выбрано:', selected);
        modal.close();
      }
    }
  ]
});
```

## ЗАМЕЧАНИЯ

- Методы `alert`, `confirm`, `prompt` являются асинхронными и возвращают `Promise`. При использовании в синхронном контексте необходимо применять `await` или `.then()`.
- Динамический импорт `i18n` внутри каждого метода означает, что локализация загружается только при необходимости, но это создаёт небольшую задержку перед отображением первого диалога.
- При закрытии окна через клик на overlay (затемнённую область) поведение эквивалентно нажатию первой кнопки из массива `buttons`.
- Одновременно может быть открыто только одно модальное окно – вызов нового автоматически закрывает предыдущее.
- Стили внедряются глобально и влияют на все элементы с соответствующими классами во всём документе.
- Для корректной работы анимации и фокуса ввода используются задержки `setTimeout(..., 10)` и `setTimeout(..., 50)`, что следует учитывать при написании тестов.

## СМ. ТАКЖЕ

- `LanguageManager` (`src/i18n/LanguageManager.js`) – система интернационализации для локализации кнопок.
- `main.js` – место переопределения глобальных функций `alert`, `confirm`, `prompt`.
- `ConstantNode.onCreate` – пример использования `modal.prompt` при создании узла через диалог.
