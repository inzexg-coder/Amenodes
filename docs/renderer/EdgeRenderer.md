# EdgeRenderer

## ОПИСАНИЕ

`EdgeRenderer` – служебный класс рендерера Amenodes, отвечающий за визуализацию связей (рёбер) между узлами графа. 
Класс создаёт SVG-элементы, вычисляет точки соединения на границах узлов и обрабатывает интерактивное удаление связей через контекстное меню (правую кнопку мыши).

**Основные возможности:**

- отрисовка всех рёбер графа в виде линий со стрелками;
- автоматический расчёт точек соединения на границах прямоугольных узлов;
- поддержка цветовой дифференциации портов (основной порт — оранжевый, специальный порт `unmapped` — голубой);
- обработка события `contextmenu` на линиях для удаления связи;
- создание переиспользуемого SVG-слоя.

**Важно:** `EdgeRenderer` не хранит состояние графа и не выполняет логических проверок связей — все данные о существующих рёбрах передаются извне.

## ЗАВИСИМОСТИ

`EdgeRenderer` не импортирует другие модули приложения и является самостоятельным классом.

# КЛАСС EDGERENDERER

## Конструктор

```javascript
constructor(layer: HTMLElement)
```

Создаёт экземпляр рендерера рёбер, привязывая его к DOM-элементу, который будет содержать SVG-слой.

**Параметры:**

- `layer` – DOM-элемент (обычно `nodesLayer`), в который будут добавляться SVG-элементы с линиями связей.

**Инициализируемые свойства:**

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.layer` | `HTMLElement` | `layer` | Контейнер для SVG-элементов. |
| `this.onEdgeRemoved` | `Function` | `null` | Коллбэк, вызываемый после удаления ребра (устанавливается через `setOnEdgeRemoved`). |

### Публичные свойства

| Свойство | Тип | Доступ | Описание |
|----------|-----|--------|----------|
| `onEdgeRemoved` | `Function | null` | чтение/запись | Функция без параметров, вызываемая после удаления ребра. |

# Методы

## renderEdges(edges, graph, rectCache)

```javascript
renderEdges(edges: Array<Edge>, graph: Graph, rectCache: Map<number, {x: number, y: number, w: number, h: number}>): void
```

Основной метод класса. Отрисовывает все переданные рёбра в SVG-слое. При каждом вызове предыдущий слой полностью заменяется новым.

**Параметры:**

- `edges` – массив объектов `Edge`, подлежащих отрисовке.
- `graph` – экземпляр `Graph`, используемый для получения узлов по их идентификаторам.
- `rectCache` – карта, где ключ — `id` узла, значение — объект с координатами и размерами узла:
  ```javascript
  {
    x: number,  // координата X узла в мировых координатах
    y: number,  // координата Y узла в мировых координатах
    w: number,  // ширина узла (всегда 280 пикселей)
    h: number   // высота узла (вычисляется через getMinHeight)
  }
  ```

**Алгоритм работы:**

1. Находит SVG-слой с классом `edge-layer` внутри `this.layer`. Если слой отсутствует — завершает работу.
2. Для каждого ребра в `edges`:
   - Получает узлы источника и приёмника через `graph.getNode()`.
   - Если любой из узлов отсутствует, ребро пропускается.
   - Получает прямоугольники узлов из `rectCache` (если отсутствуют — пропускает).
   - Вычисляет точки на границах узлов через `getBorderPoint()`.
   - **Валидация координат:** если `getBorderPoint()` вернул `NaN` или `Infinity` (невалидная геометрия), ребро пропускается — линия остаётся на предыдущей валидной позиции, что предотвращает «обрыв» ребра при резких движениях узла.
   - Обновляет атрибуты SVG-линии (`x1`, `y1`, `x2`, `y2`).
   - Обновляет атрибуты SVG-стрелки (`points`).

1. Удаляет предыдущий SVG-слой с классом `edge-layer`, если он существует.
2. Создаёт новый SVG-элемент через `createSvgLayer()`.
3. Для каждого ребра в `edges`:
   - Получает узлы источника и приёмника через `graph.getNode()`.
   - Если любой из узлов отсутствует, ребро пропускается.
   - Получает прямоугольники узлов из `rectCache` (если отсутствуют — пропускает).
   - Вычисляет точки на границах узлов через `getBorderPoint()`.
   - Определяет цвет линии в зависимости от порта: `#44aaff` для `unmapped`, `#ffb347` для остальных.
   - Создаёт линию через `createLine()` и стрелку через `createArrow()`.
   - Добавляет оба элемента в SVG.
   - Навешивает обработчик `contextmenu` на линию для удаления ребра.
4. Добавляет SVG-слой в `this.layer`.

## createSvgLayer()

```javascript
createSvgLayer(): SVGSVGElement
```

Создаёт и возвращает новый SVG-элемент, настроенный для отображения рёбер.

**Возвращает:** `SVGSVGElement` со следующими атрибутами:

| Атрибут | Значение | Описание |
|---------|----------|----------|
| `class` | `'edge-layer'` | Идентификация слоя для последующего удаления. |
| `width` | `'100%'` | Растяжение на всю ширину родителя. |
| `height` | `'100%'` | Растяжение на всю высоту родителя. |
| `style.position` | `'absolute'` | Абсолютное позиционирование внутри контейнера. |
| `style.top/left` | `'0'` | Привязка к левому верхнему углу. |
| `style.pointerEvents` | `'none'` | Запрет перехвата событий мыши (кроме линий, у которых `pointerEvents` устанавливается отдельно). |
| `style.overflow` | `'visible'` | Отображение элементов, выходящих за границы SVG. |

## createLine(p1, p2, color, edgeId)

```javascript
createLine(p1: {x: number, y: number}, p2: {x: number, y: number}, color: string, edgeId: number): SVGLineElement
```

Создаёт SVG-линию между двумя точками.

**Параметры:**

- `p1` – объект с координатами `x`, `y` начальной точки.
- `p2` – объект с координатами `x`, `y` конечной точки.
- `color` – цвет линии в формате, поддерживаемом SVG (например, `#ffb347`).
- `edgeId` – идентификатор ребра, сохраняемый в атрибуте `data-edge-id`.

**Возвращает:** `SVGLineElement` со следующими атрибутами:

| Атрибут | Значение | Описание |
|---------|----------|----------|
| `x1`, `y1`, `x2`, `y2` | координаты | Позиция линии. |
| `stroke-width` | `'3'` | Толщина линии. |
| `stroke-linecap` | `'round'` | Скругление концов. |
| `class` | `'edge-line'` | CSS-класс для стилизации (наведение изменяет толщину). |
| `stroke` | `color` | Цвет линии. |
| `style.pointerEvents` | `'visibleStroke'` | Линия получает события мыши только при наведении на саму линию (не на прозрачную область). |
| `data-edge-id` | `edgeId` | Идентификатор для отладки. |

## createArrow(p1, p2, color)

```javascript
createArrow(p1: {x: number, y: number}, p2: {x: number, y: number}, color: string): SVGPolygonElement
```

Создаёт стрелку в виде полигона, расположенную на середине линии, указывающую направление от `p1` к `p2`.

**Параметры:**

- `p1` – начальная точка линии.
- `p2` – конечная точка линии.
- `color` – цвет стрелки (совпадает с цветом линии).

**Алгоритм построения стрелки:**

1. Вычисляет среднюю точку отрезка: `midX = (p1.x + p2.x) / 2`, `midY = (p1.y + p2.y) / 2`.
2. Вычисляет угол направления линии: `angle = atan2(p2.y - p1.y, p2.x - p1.x)`.
3. Определяет точку острия стрелки (на 6 пикселей дальше средней точки по направлению линии).
4. Определяет точку основания стрелки (на 8 пикселей назад от средней точки).
5. Определяет боковые точки для создания треугольной формы (отклонение на 5 пикселей перпендикулярно линии).
6. Формирует строку координат для полигона.

**Возвращает:** `SVGPolygonElement` со следующими атрибутами:

| Атрибут | Значение | Описание |
|---------|----------|----------|
| `points` | строка координат | Три точки полигона (остриё и два основания). |
| `fill` | `color` | Заливка цветом линии. |
| `stroke` | `color === '#44aaff' ? '#88ccff' : '#ffda99'` | Обводка более светлым оттенком для контраста. |
| `stroke-width` | `'1'` | Толщина обводки. |
| `stroke-linejoin` | `'round'` | Скругление углов полигона. |

## getBorderPoint(fromRect, toRect)

```javascript
getBorderPoint(fromRect: {x: number, y: number, w: number, h: number}, toRect: {x: number, y: number, w: number, h: number}): {x: number, y: number}
```

Вычисляет точку на границе прямоугольника `fromRect`, через которую проходит прямая линия к центру прямоугольника `toRect`. Используется для соединения узлов линиями, начинающимися на границе, а не в центре.

**Параметры:**

- `fromRect` – прямоугольник узла-источника с полями `x`, `y`, `w`, `h`.
- `toRect` – прямоугольник узла-приёмника с полями `x`, `y`, `w`, `h`.

**Алгоритм:**

1. Вычисляет центры обоих прямоугольников.
2. Вычисляет вектор от центра `fromRect` к центру `toRect` (`dx`, `dy`).
3. Если вектор нулевой (прямоугольники совпадают по центру), возвращает точку на нижней границе `fromRect`.
4. Вычисляет параметры `t` для пересечения с четырьмя гранями прямоугольника:
   - Левая грань (`x = fromRect.x`)
   - Правая грань (`x = fromRect.x + fromRect.w`)
   - Верхняя грань (`y = fromRect.y`)
   - Нижняя грань (`y = fromRect.y + fromRect.h`)
5. Для каждой грани вычисляет `t` (параметр вдоль вектора) и проверяет, что точка пересечения попадает в пределы грани.
6. Выбирает наименьшее положительное значение `t` (ближайшее пересечение).
7. Возвращает координаты точки пересечения.

**Возвращает:** объект `{x, y}` с координатами точки на границе `fromRect`.

## setOnEdgeRemoved(callback)

```javascript
setOnEdgeRemoved(callback: Function): void
```

Устанавливает коллбэк-функцию, которая будет вызываться после удаления ребра (в обработчике `contextmenu`).

**Параметры:**

- `callback` – функция без параметров, вызываемая после удаления ребра.

**Пример использования:**

```javascript
edgeRenderer.setOnEdgeRemoved(() => {
  graph.reevaluateAll();
  graph.updateAllOutputs();
  domRenderer.render();
  domRenderer.save();
});
```

# ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Базовое использование в DomRenderer

```javascript
// В конструкторе DomRenderer
this.edgeRenderer = new EdgeRenderer(this.layer);
this.edgeRenderer.setOnEdgeRemoved(() => {
  this.graph.reevaluateAll();
  this.graph.updateAllOutputs();
  this.render();
  this.save();
});

// В методе render()
const rectCache = new Map();
for (const node of visibleNodes) {
  rectCache.set(node.id, {
    x: node.x,
    y: node.y,
    w: 280,
    h: this.getNodeHeight(node)
  });
}
this.edgeRenderer.renderEdges(this.graph.edges, this.graph, rectCache);
```

### Самостоятельное использование

```javascript
import { EdgeRenderer } from './renderer/EdgeRenderer.js';

const layer = document.getElementById('nodesLayer');
const renderer = new EdgeRenderer(layer);

// В цикле анимации
function renderEdges() {
  const rects = new Map();
  graph.nodes.forEach(node => {
    rects.set(node.id, { x: node.x, y: node.y, w: 280, h: node.getMinHeight() });
  });
  renderer.renderEdges(graph.edges, graph, rects);
}

renderer.setOnEdgeRemoved(() => console.log('Edge was deleted'));
```


## Премиум-функции

### Edge Wave (анимация на рёбрах)

Гейтинг: класс `premium-edge-wave` на `document.body` при `localStorage('premium_edge_wave') === 'true'` и `_isPremium()`.

### Hover-эффект

```css
.premium-edge-wave .edge-group:hover .edge-line {
  stroke-width: 5;
  stroke-dasharray: 10 8;
  animation: edge-wave 0.5s linear infinite;
  filter: brightness(1.8) drop-shadow(0 0 6px var(--accent));
}
```

При наведении на ребро линия становится пунктирной с анимацией `stroke-dashoffset`.

### Touch-эффект

На мобильных устройствах при касании ребра добавляется класс `.edge-wave` на 700мс:

```javascript
group.addEventListener('touchstart', () => {
  group.classList.add('edge-wave');
  setTimeout(() => group.classList.remove('edge-wave'), 700);
});
```

CSS для touch:

```css
.premium-edge-wave .edge-group.edge-wave .edge-line {
  stroke-dasharray: 12 8;
  animation: edge-wave 0.4s linear infinite;
}
```

### Click-эффект

Аналогично touch, но на `click`. Стрелка и линия получают свечение `drop-shadow`.


### Магнитные узлы и стрелка предпросмотра

Гейтинг: `_isPremium() && localStorage('premium_magnetic_nodes') === 'true'`. При перетаскивании связи рядом с совместимым узлом отрисовывается временная стрелка (см. `DomRenderer._addTempArrow`). Создаётся в `DomRenderer`, так как отображается только во время drag.


# ЗАМЕЧАНИЯ

- Метод `getBorderPoint` предполагает, что размер узла фиксирован по ширине (280 пикселей) и вычисляется по высоте через `getMinHeight`. При изменении парадигмы отрисовки узлов этот метод может давать неверные результаты.
- Стрелка всегда рисуется **на середине** линии, а не на конце. Это сделано для того, чтобы стрелка не перекрывалась с узлом-приёмником при коротких связях.
- Событие `contextmenu` на линии не вызывает всплытие, поэтому удаление ребра происходит без открытия контекстного меню узла.
- При удалении ребра через `contextmenu` **не вызывается** автоматический пересчёт графа или перерисовка — это должно быть реализовано в коллбэке `onEdgeRemoved`.
- SVG-элементы не кэшируются: при каждом вызове `renderEdges` старый слой полностью удаляется и создаётся заново. Для графов с тысячами рёбер это может быть затратно.
