# `FPSCounter`

## ОПИСАНИЕ

`FPSCounter` – утилитарный класс для измерения частоты кадров (Frames Per Second) в браузере. Предоставляет два режима работы:

- непрерывный мониторинг FPS с отображением в указанном DOM-элементе;
- одноразовое измерение средней частоты кадров за заданный промежуток времени.

Класс использует `requestAnimationFrame` для синхронизации с циклом отрисовки браузера, что обеспечивает максимальную точность измерений, релевантную для визуальных приложений.

**Важно:** класс не зависит от других модулей Amenodes и может использоваться автономно в любом веб-приложении.

## ЗАВИСИМОСТИ

`FPSCounter` не импортирует другие модули и не имеет внешних зависимостей.

# КЛАСС `FPSCounter`

## Конструктор

```javascript
constructor(elementId: string)
```

Создаёт экземпляр счётчика FPS и автоматически запускает непрерывный мониторинг.

**Параметры:**

- `elementId` – строковый идентификатор DOM-элемента, в который будет выводиться текущее значение FPS. Элемент должен существовать на момент вызова конструктора.

**Инициализация:**

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.element` | `HTMLElement \| null` | результат `document.getElementById(elementId)` | Элемент для отображения FPS. |
| `this.lastTime` | `number` | `performance.now()` | Метка времени последнего обновления счётчика. |
| `this.frameCount` | `number` | `0` | Счётчик кадров, отрисованных с момента последнего обновления. |

**Пример:**

```javascript
// HTML должен содержать элемент: <div id="fpsMeter"></div>
const fps = new FPSCounter('fpsMeter');
// В элементе начнёт отображаться "FPS: 60" (или текущее значение)
```

# Публичные методы

## start()

```javascript
start(): void
```

Запускает бесконечный цикл измерения и отображения FPS. Метод вызывается автоматически в конструкторе – ручной вызов может быть использован для перезапуска после остановки.

**Алгоритм:**

1. Регистрирует рекурсивный вызов `requestAnimationFrame`.
2. При каждом кадре инкрементирует `this.frameCount`.
3. Каждую секунду вычисляет FPS по формуле:

   ```
   fps = round(frameCount * 1000 / (currentTime - lastTime))
   ```

4. Обновляет `textContent` элемента, если он существует, в формате `"FPS: {fps}"`.
5. Сбрасывает счётчики и продолжает цикл.

**Примечание:** метод не возвращает управление – цикл продолжается до закрытия страницы или явного уничтожения объекта.

**Пример:**

```javascript
const fps = new FPSCounter('fpsMeter');
// Мониторинг уже запущен, start() вызван автоматически.

// Если по какой-то причине цикл прерван:
fps.start(); // перезапуск
```

## measure(durationMs)

```javascript
measure(durationMs: number = 1500): Promise<number>
```

Выполняет одноразовое измерение средней частоты кадров за указанный интервал времени. Метод асинхронный, возвращает `Promise`, разрешающийся после завершения измерения.

**Параметры:**

- `durationMs` – длительность измерения в миллисекундах. По умолчанию `1500` (1.5 секунды). Должен быть положительным числом.

**Алгоритм:**

1. Запоминает начальное время через `performance.now()`.
2. Устанавливает внутренний счётчик кадров в `0`.
3. Запускает рекурсивный цикл `requestAnimationFrame`, инкрементирующий счётчик при каждом кадре.
4. По достижении заданной длительности вычисляет FPS по формуле:

   ```
   fps = round(frames * 1000 / (endTime - startTime))
   ```

5. Разрешает `Promise` с полученным значением.

**Возвращает:** `Promise<number>` – округлённое целое значение FPS.

**Важно:** измерение не влияет на непрерывный мониторинг (если он был запущен через конструктор). Два параллельных цикла `requestAnimationFrame` будут работать независимо.

**Пример:**

```javascript
const fps = new FPSCounter('fpsMeter');

async function benchmark() {
  const averageFps = await fps.measure(2000);
  console.log(`Средний FPS за 2 секунды: ${averageFps}`);
  
  if (averageFps < 30) {
    console.warn('Производительность ниже рекомендуемой');
  }
}

benchmark();
```

# Внутренние механизмы

## Синхронизация с `requestAnimationFrame`

Оба режима работы используют `requestAnimationFrame` – браузерный API, вызывающий колбэк перед каждой перерисовкой. Это обеспечивает:

- измерение именно визуальной частоты кадров;
- автоматическую паузу при неактивной вкладке (fps падает до 0);
- энергоэффективность по сравнению с `setInterval`.

### Формула расчёта

```
fps = round(frameCount * 1000 / deltaTime)
```

где `deltaTime` – разница между текущим временем и временем последнего обновления в миллисекундах.

# ПРИМЕР ИСПОЛЬЗОВАНИЯ

```javascript
import { FPSCounter } from './utils/FPSCounter.js';

// Создание элемента в HTML (добавляется динамически)
const fpsDiv = document.createElement('div');
fpsDiv.id = 'myFpsCounter';
document.body.appendChild(fpsDiv);

// Инициализация непрерывного мониторинга
const monitor = new FPSCounter('myFpsCounter');

// Асинхронное измерение производительности тяжёлой операции
async function measureHeavyOperation() {
  // Замер FPS до операции
  const beforeFps = await monitor.measure(1000);
  console.log(`FPS до: ${beforeFps}`);
  
  // Симуляция тяжёлой работы
  for (let i = 0; i < 1000000; i++) {
    Math.sqrt(i);
  }
  
  // Замер после
  const afterFps = await monitor.measure(1000);
  console.log(`FPS после: ${afterFps}`);
  
  if (afterFps < beforeFps * 0.7) {
    console.warn('Производительность значительно снижена');
  }
}

measureHeavyOperation();
```

# ОШИБКИ И ОГРАНИЧЕНИЯ

| Ситуация | Поведение |
|----------|-----------|
| Элемент с указанным `elementId` не найден | `this.element` устанавливается в `null`, непрерывный мониторинг работает без вывода (ошибки не возникает). |
| `durationMs` передан как `0` или отрицательное число | Измерение выполнится, но `deltaTime` может быть близка к 0, что приведёт к некорректно высокому значению FPS. Рекомендуется использовать значения ≥ 500. |
| Вкладка неактивна во время измерения | `requestAnimationFrame` приостанавливается, измеренное значение FPS будет близко к 0 (что соответствует реальному поведению). |

## ПРИМЕЧАНИЯ ПО ПРОИЗВОДИТЕЛЬНОСТИ

- Непрерывный мониторинг добавляет крайне незначительную нагрузку – один вызов `requestAnimationFrame` и несколько арифметических операций в секунду.
- Параллельные вызовы `measure` и непрерывный мониторинг используют отдельные циклы `requestAnimationFrame`, что корректно, но создаёт два независимых колбэка на каждый кадр.
- Для высокоточных измерений (например, сравнения оптимизаций) рекомендуется использовать `measure` с длительностью не менее 1000 мс для усреднения пульсаций.
