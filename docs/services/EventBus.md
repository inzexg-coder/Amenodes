# `EventBus`

## ОПИСАНИЕ

`EventBus` – легковесная шина событий (pub/sub), реализующая паттерн «издатель-подписчик» для обеспечения слабой связанности между компонентами приложения Amenodes. Класс позволяет:

- регистрировать обработчики событий (подписка);
- отправлять события с произвольными данными (публикация);
- отменять подписку на события.

## ЗАВИСИМОСТИ

Класс `EventBus` не импортирует внешние модули. Реализация использует только нативные возможности JavaScript.

# КЛАСС EVENTBUS

## Конструктор

```javascript
constructor()
```

Создаёт новый экземпляр шины событий. Инициализирует:

| Свойство | Тип | Начальное значение | Описание |
|----------|-----|--------------------|----------|
| `this.listeners` | `Map<string, Array<Function>>` | `new Map()` | Хранилище подписчиков, где ключ — имя события, значение — массив callback-функций. |

### Публичные свойства

Класс `EventBus` не предоставляет публичных свойств для прямого доступа. Вся работа осуществляется через методы.

# Методы

## on(event, callback)

```javascript
on(event: string, callback: Function): void
```

Регистрирует функцию-обработчик для указанного события. Если для данного события ещё не существует массива подписчиков, он создаётся автоматически. 
Несколько обработчиков могут быть зарегистрированы на одно событие; порядок их вызова соответствует порядку регистрации.

**Параметры:**

- `event` – строковый идентификатор события (например, `'node:added'`, `'graph:changed'`).
- `callback` – функция, которая будет вызвана при наступлении события. Функция принимает один аргумент — данные, переданные при вызове `emit`.

**Побочные эффекты:**

- Обновляется внутренняя карта `listeners`: для ключа `event` создаётся или дополняется массив колбэков.

**Пример:**

```javascript
const bus = new EventBus();

bus.on('node:selected', (nodeId) => {
  console.log(`Выбран узел ${nodeId}`);
});

bus.on('graph:changed', () => {
  console.log('Граф был изменён');
});

bus.on('custom:event', (data) => {
  console.log('Получены данные:', data);
});
```

## off(event, callback)

```javascript
off(event: string, callback: Function): void
```

Удаляет ранее зарегистрированный обработчик для указанного события. Если для события не существует подписчиков или переданный callback не найден в массиве, метод ничего не делает.

**Параметры:**

- `event` – строковый идентификатор события.
- `callback` – функция-обработчик, которая была передана в `on()`.

- Удаляет указанный callback из массива подписчиков события.
- Если после удаления массив становится пустым, запись в `listeners` сохраняется (остаётся пустой массив). Полное удаление ключа не производится.

**Пример:**

```javascript
const bus = new EventBus();

const handler = (data) => console.log(data);
bus.on('test', handler);

// Позже:
bus.off('test', handler); // обработчик больше не будет вызываться
```

## emit(event, data)

```javascript
emit(event: string, data?: any): void
```

Инициирует событие, вызывая все зарегистрированные для него функции-обработчики. Каждый обработчик вызывается синхронно с переданными данными. Если для события не зарегистрировано ни одного обработчика, метод завершается без ошибок.

**Параметры:**

- `event` – строковый идентификатор события.
- `data` – (опционально) произвольные данные, которые будут переданы каждому обработчику. Может быть любого типа: число, строка, объект, массив, `null`, `undefined`.

- Синхронный вызов всех callback-функций, зарегистрированных для события.
- Порядок вызова соответствует порядку регистрации через `on()`.

**Пример:**

```javascript
const bus = new EventBus();

bus.on('user:login', (username) => {
  console.log(`Пользователь ${username} вошёл в систему`);
});

bus.on('user:login', () => {
  console.log('Обновляем интерфейс...');
});

bus.emit('user:login', 'alice');
// Вывод:
// Пользователь alice вошёл в систему
// Обновляем интерфейс...
```

**Примечание:** Если обработчик выбрасывает исключение, оно не перехватывается `EventBus` и всплывает наружу. Рекомендуется использовать `try/catch` внутри обработчиков при необходимости.

# ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ

### Базовый сценарий: уведомление об изменении графа

```javascript
import { EventBus } from './services/EventBus.js';

const eventBus = new EventBus();

// Компонент рендерера подписывается на изменения
eventBus.on('graph:updated', (graphData) => {
  renderer.render(graphData);
});

// Компонент истории подписывается на сохранение
eventBus.on('graph:updated', () => {
  history.save();
});

// Где-то в контроллере после изменения графа:
function addNodeAndNotify(node) {
  graph.addNode(node);
  eventBus.emit('graph:updated', graph.toSerial());
}
```

### Сценарий: коммуникация между несвязанными модулями

```javascript
// Модуль A
eventBus.on('optimization:toggle', ({ name, enabled }) => {
  console.log(`Оптимизация ${name} ${enabled ? 'включена' : 'выключена'}`);
  applyOptimization(name, enabled);
});

// Модуль B
function onOptimizationPanelChange() {
  eventBus.emit('optimization:toggle', {
    name: 'virtualization',
    enabled: true
  });
}
```

### Сценарий: множественные подписки с разными данными

```javascript
const bus = new EventBus();

bus.on('node:added', (node) => {
  console.log(`Добавлен узел: ${node.type} (id=${node.id})`);
});

bus.on('node:added', () => {
  updateNodeCounter();
});

bus.on('node:added', () => {
  eventBus.emit('graph:changed'); // цепочка событий
});

bus.emit('node:added', { id: 5, type: 'calc', x: 100, y: 200 });
```

### Сценарий: отмена подписки для предотвращения утечек памяти

```javascript
class Component {
  constructor(eventBus) {
    this.eventBus = eventBus;
    this.handler = this.onData.bind(this);
    this.eventBus.on('data:ready', this.handler);
  }
  
  onData(data) {
    this.process(data);
  }
  
  destroy() {
    // Важно: отписываемся при уничтожении компонента
    this.eventBus.off('data:ready', this.handler);
  }
}
```

## ЗАМЕЧАНИЯ

- **Порядок вызова обработчиков** гарантированно соответствует порядку их регистрации.
- **Отсутствие проверки типов:** метод `emit` не проверяет соответствие типов данных ожиданиям обработчиков.
- **Синхронность:** все обработчики вызываются синхронно в момент вызова `emit`. Асинхронные операции внутри обработчиков должны управляться самостоятельно (через `setTimeout`, `Promise` и т.д.).
- **Отсутствие wildcard-событий:** `EventBus` не поддерживает шаблоны событий (например, `'user:*'`). Каждое событие идентифицируется точной строкой.
- **Отсутствие приоритетов:** нельзя задать приоритет выполнения обработчиков.
- **Отсутствие одноразовых обработчиков:** для реализации поведения «вызвать один раз» необходимо вручную отписываться внутри обработчика.
- **Утечки памяти:** при длительном существовании приложения важно отписываться от событий в методах `destroy` или `onDetach` компонентов, иначе обработчики продолжат существовать в памяти.

## СРАВНЕНИЕ С АЛЬТЕРНАТИВАМИ

| Подход | Преимущества | Недостатки |
|--------|--------------|-------------|
| `EventBus` | Простота, независимость, явная коммуникация | Нет типизации, ручное управление подписками |
| Прямой вызов методов | Высокая производительность, типизация | Жёсткая связанность компонентов |
| Глобальные переменные | Предельная простота | Антипаттерн, сложность отладки |
| `window.dispatchEvent` | Встроенный механизм браузера | Работает только в DOM-контексте |
